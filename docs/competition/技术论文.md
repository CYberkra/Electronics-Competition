# 基于 JESD204B 的高速任意波形信号发生器设计

## 摘要

本文设计并实现了一款基于 Xilinx Kintex-7 FPGA 和 AD9144 高速数模转换器的任意波形信号发生器（AWG）。系统采用 JESD204B 高速串行接口实现 FPGA 与 DAC 之间的数据传输，支持 4 通道 16 bit、2.8 GSa/s 采样率的任意波形输出。数字基带部分使用 48 bit 相位累加器实现直接数字频率合成（DDS），频率分辨率达到微赫兹量级，支持正弦、方波、三角、锯齿四种标准波形以及基于 BRAM 的任意波形回放。系统集成了硬件扫频引擎、UART 远程控制、板载按键交互等完整功能，并设计了搭载 ST7789 显示屏和 EC11 旋转编码器的扩展模块以满足赛题 PCB 丝印要求。实测表明系统在 100 kHz 至 300 MHz 范围内可稳定输出正弦波，验证了 JESD204B 高速链路和 DDS 数字基带的正确性。

**关键词**：任意波形发生器；JESD204B；FPGA；DDS；AD9144；高速串行接口

## 1 引言

任意波形发生器是电子测试测量领域的核心仪器，广泛应用于通信系统测试、雷达信号仿真、集成电路验证和量子计算控制等场景。目前高速高精度 AWG 市场主要被 Keysight、Tektronix 等国外厂商垄断，国产化替代需求迫切。

第二十一届中国研究生电子设计竞赛优利德命题赛题二要求"根据 DDS 原理，通过嵌入式软硬件协同设计，设计一个简易任意波形发生器"，核心指标包括：采样率不低于 5 GSa/s、垂直分辨率不低于 14 bit、模拟带宽不低于 1 GHz、频率范围覆盖 1 mHz 至 1 GHz 等。

本文基于正点原子 K7-325T 开发平台和 FMCADDA-9250-9144 子卡，采用 AD9144 四通道 16 bit 2.8 GSa/s DAC，通过 JESD204B Subclass 1 高速串行接口实现 4 通道 @10 Gbps 数据传输，设计并实现了一款功能完整的任意波形信号发生器。

## 2 系统总体方案

### 2.1 硬件平台

系统硬件由三部分组成：

- **FPGA 主板**：正点原子 K7-325T 开发板，搭载 Xilinx Kintex-7 XC7K325TFFG900-2，提供 16 个 GTX 高速收发器和 FMC HPC 连接器。
- **FMC 子卡**：Analog Devices FMCADDA-9250-9144，集成 AD9144 四通道 16 bit 2.8 GSa/s DAC 和 LMK04828 双 PLL 时钟芯片。
- **扩展模块**：自研 KiCad PCB，搭载 ST7789 1.69" 显示屏和 EC11 旋转编码器，丝印层印制研电赛 Logo 及日期以满足赛题硬性要求。

### 2.2 系统架构

系统采用 FPGA 数字基带 + JESD204B 高速链路 + DAC 模拟输出的三层架构：

- **数字基带层**：48 bit DDS 引擎在 FPGA 内部实时生成 4 路并行 16 bit 波形样点（4 采样/周期，满足 JESD204B 8 字节/帧的帧格式要求），经过 Q1.15 幅度缩放、波形选择和数字校准后，通过 sample_packer 模块打包为 128 bit 并行数据。
- **高速链路层**：JESD204B TX Core 将并行数据封装为 8B/10B 编码的串行帧，通过 4 对 GTX 收发器（每路 10.3125 Gbps 线速率）发送至 AD9144 DAC。
- **模拟输出层**：AD9144 接收 JESD204B 数据流，经内部 4 倍插值和数模转换后输出四路差分电流信号，经板载变压器转换为单端 50 Ω 输出。

### 2.3 时钟系统

LMK04828 双 PLL 时钟芯片以板载 50 MHz TCXO 为参考源，为系统提供全部同源时钟：

| 时钟信号 | 频率 | 用途 |
|---------|------|------|
| DACCLK | 2.8 GHz | AD9144 采样时钟 |
| GTX REFCLK | 125 MHz | FPGA GTX QPLL 参考时钟 |
| SYSREF | 可编程 | JESD204B Subclass 1 确定性延迟同步 |
| glblclk | 125 MHz | FPGA 内部 MMCM 输入 → tx_core_clk (250 MHz) |

同源时钟设计保证了 FPGA-DAC 链路的严格时钟同步，避免了跨时钟域的数据丢失或重复问题。

## 3 关键技术

### 3.1 高精度 DDS 引擎

直接数字频率合成（DDS）是本系统的核心。DDS 引擎采用 48 bit 相位累加器，每个 tx_core_clk 周期（250 MHz）并行产生 4 个采样点，等效 DAC 数据率为 1 GSa/s，配合 AD9144 内部 4 倍插值实现 2.8 GSa/s 输出。

频率控制字计算公式为：

> Δθ = f_out × 2^48 / f_sample

其中 f_sample = 1 GSa/s（等效），频率分辨率为 1 × 10^9 / 2^48 ≈ 3.55 μHz，远超赛题 1 mHz 要求。

四种波形通过不同路径生成：
- **正弦波**：相位累加器高 12 位索引 4096 深度 Sine LUT（1/4 周期压缩存储，利用对称性恢复完整周期），输出 16 bit 有符号样点。
- **方波**：相位最高位直接判决输出 ± 满量程，零 LUT 开销。
- **三角波**：相位到幅度的线性映射 + 对称折叠。
- **锯齿波**：相位直接映射为幅度。

为改善时序收敛，DDS 引擎采用 4 级流水线架构：第 1 级 LUT 读取 → 第 2 级形状计算 → 第 3 级 DSP48 乘法 → 第 4 级饱和输出。

### 3.2 JESD204B 高速串行接口

JESD204B 是本系统完成 FPGA 到 DAC 高速数据传输的关键技术。链路配置如下：

| 参数 | 值 | 说明 |
|------|----|------|
| L（通道数） | 4 | GTX 收发器数量 |
| M（转换器数） | 4 | DAC 通道数 |
| F（每帧字节数） | 4 | 4 个 16-bit 样点 |
| S（每转换器每帧采样数） | 1 | |
| K（每多帧的帧数） | 32 | |
| 线速率 | 10.3125 Gbps | 8B/10B 编码 |

采用 Subclass 1 模式实现确定性延迟：LMK04828 产生的 SYSREF 信号作为系统全局时间基准，通过等长布线（长度差 < 5 mil）保证 FPGA 与 DAC 的 SYSREF 到达时间一致。

### 3.3 扫频引擎

硬件扫频引擎（sweep_engine.v）是一个有限状态机，按设定的步进频率（Δf）和驻留时间（Δt）自动更新 DDS 频率控制字，无需 CPU 干预。由于 DDS 是流水线结构，频率跳变延迟仅为数个时钟周期，扫频速率可达微秒级步进间隔。

### 3.4 控制与交互

系统提供三种控制方式：
- **UART 远程控制**：115200 8N1 ASCII 协议，支持读写 20+ 配置寄存器，可实现频率、幅度、波形、扫频参数的全功能远程配置。
- **板载按键**：KEY0 切换参数页（频率/波形/幅度/偏置），KEY1 循环预设值。
- **扩展模块**（待开发）：ST7789 显示屏 + EC11 旋转编码器，提供本地可视化交互。

UART 控制需先写入控制寄存器 bit1 = 1 使能 UART 控制源，否则按键值优先级更高。

## 4 测试结果

### 4.1 测试环境

- 示波器：Agilent DSO-X 3034A（350 MHz 带宽，4 GSa/s）
- 测试条件：50 Ω 端接，幅度设置 0x7FFF（满量程），正弦波输出
- 固件版本：20260507

### 4.2 幅度-频率特性

| 频率 | Vpp | 备注 |
|------|-----|------|
| 100 kHz | 60 mV | 输出变压器低频截止 |
| 1 MHz | 120 mV | |
| 10 MHz | 500 mV | |
| 20 MHz | 640 mV | 谐振峰 |
| 50 MHz | 520 mV | |
| 100 MHz | 680 mV | 最大响应点 |
| 200 MHz | 560 mV | |
| 300 MHz | 470 mV | 高频滚降 |

### 4.3 结果分析

实测带内平坦度 > 20 dB，不满足赛题 < 3 dB 要求。主要原因是：
1. FMC 子卡输出变压器为交流耦合，低频响应差；
2. 无外部宽带输出放大器，DAC 直接驱动 50 Ω 负载，最大输出仅约 1 Vpp；
3. 数字校准表尚未实例化，无法补偿频率相关的幅度不平坦。

可通过增加外部 RF 宽带放大器 + 启用数字校准表 + AD9144 反 sinc 滤波器来显著改善平坦度。

## 5 结论

本文基于 FPGA + JESD204B + AD9144 技术路线设计并实现了一款功能完整的任意波形信号发生器。系统在垂直分辨率（16 bit）、频率分辨率（μHz 级）和通道数（4 路）方面超过赛题指标要求，在采样率（2.8 GSa/s vs 5 GSa/s 要求）、输出幅度（最大 ~1 Vpp vs 3 Vpp 要求）和带内平坦度（>20 dB vs <3 dB 要求）方面与赛题存在差距。差距根因为硬件平台的物理限制（DAC 采样率上限、输出变压器带宽、无外部放大器），可通过升级 DAC 器件、增加输出放大器和启用数字校准来弥补。

系统的核心贡献在于：
1. 完整实现了 JESD204B Subclass 1 高速链路（4 L @10 Gbps）；
2. 48 bit DDS 引擎达到 μHz 级频率分辨率，4 级流水线架构保证时序收敛；
3. 硬件扫频引擎、UART 远程控制、SPI 统一配置框架等功能完备；
4. SPI 控制器完成了 derived-clock → clock-enable 的设计规范化改进。
