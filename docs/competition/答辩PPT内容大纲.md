## Slide 3: 项目背景与赛题解读

**内容（分栏布局）：**

**左栏 — 应用背景：**
- 任意波形发生器(AWG)是电子测试测量领域的核心仪器，广泛用于通信系统测试、雷达信号仿真、IC测试、量子计算控制等场景
- 高速高精度AWG长期被Keysight、Tektronix等国外厂商垄断，国产化替代需求迫切
- 赛题要求设计一款采样率≥5Gsps、垂直分辨率≥14bit的多通道任意波形信号发生器

**右栏 — 赛题指标解读（关键指标提取）：**
| 指标项 | 赛题要求 | 技术含义 |

| 任意波形 | 支持 | 非周期信号输出 |

| 控制接口 | 需上位机/本地控制 | 人机交互 |

**底部总结框：**
赛题本质是考察高速数字信号链(FPGA+DAC)、时钟分配、高速串行接口三大核心能力的系统工程实现

```
│  │ Kintex-7  │                                │  DAC  │─┤→ CH0

│  │           │◄──────────────────────────────►│       │─┤→ CH2

│  │  └─────┘  │   │  Clock   │    ┌──────┐               │

│  │  ┌─────┐  │   DEVCLK + SYSREF                         │
│  │  │Ctrl │  │◄───(to FPGA & DAC)                        │
│  │  └─────┘  │                                          │
│  └──────────┘                                           │
│       │                                                  │
│   按键+LED (本地UI)                                       │
└─────────────────────────────────────────────────────────┘
```

**信号流说明（分步文字）：**
1. **波形生成：** 48位相位累加器DDS在FPGA内部实时计算波形样点 → 存入JESD204B发送FIFO
2. **数据传输：** JESD204B TX Core将并行样点数据打包为8B/10B编码的串行帧 → 通过4对GTX收发器(每路10.3125Gbps)发送至AD9144
3. **数模转换：** AD9144接收JESD204B数据流，内部解帧、插值 → 4通道同步DAC输出模拟波形
4. **时钟同步：** LMK04828提供同源DEVCLK(采样时钟)和SYSREF(确定性延迟同步脉冲)，保证FPGA-DAC链路的严格时钟同步

**视觉说明：**
- 使用三种颜色区分数据流(蓝色)、时钟流(红色)、控制流(绿色)
- 每个模块配简短功能标注
- 底部配信号流步骤编号图

**演讲者备注：**
"系统采用FPGA主板+FMC子卡的分体架构。FPGA选型为Kintex-7 XC7K325T，它提供16个GTX高速收发器，完全满足4通道JESD204B的物理层需求。DAC选用ADI公司的AD9144，这是一款四通道16位2.8Gsps的JESD204B接口DAC，是我们的核心器件。时钟芯片LMK04828同时为FPGA GTX参考时钟和DAC采样时钟提供同源时钟，并通过SYSREF信号实现JESD204B Subclass 1的确定性延迟。整个信号链路的关键特征有三个：全数字域波形生成、JESD204B高速串行传输、同源时钟同步。"

## Slide 6: 核心技术1 — 高速JESD204B接口

**内容：**

**左侧 — 技术要点：**
- **协议层配置：**
  - JESD204B Subclass 1（支持确定性延迟）
  - 4 Lanes × 10.3125 Gbps 线速率
  - 8B/10B 编码（25%开销，有效带宽约8Gbps/lane）
  - 链路参数：L=4, M=4(转换器数), F=1(每帧字节数), S=1(每转换器采样数)
  - 帧格式：每帧含4个16-bit样点(4通道交错)

- **物理层配置：**
  - XC7K325T GTX收发器配置：QPLL @ 5.15625GHz (半速率模式)
  - TX Pre-emphasis 自适应均衡
  - 眼图模板满足JESD204B规范要求

- **链路建立流程：**
  - Code Group Sync (CGS) → Initial Lane Alignment (ILAS) → User Data
  - LMK04828 SYSREF脉冲触发确定性延迟校准

**右侧 — 插图占位：**
- JESD204B链路建立状态机流程图（CGS→ILAS→DATA三阶段）
- GTX眼图仿真结果占位（[插图: GTX TX Eye Diagram]）

**关键技术参数框（黄色高亮）：**
JESD204B总吞吐量：4 Lanes × 10.3125 Gbps × 0.8 (8B/10B效率) = 33 Gbps有效带宽
4通道×16bit×2.8Gsps = 179.2 Gbps 原始采样数据率
(注：AD9144内部插值和JESD204B链路层会进行数据压缩/映射)

**视觉说明：**
- 上半部分文字要点分两列
- 中部插入链路状态机示意图
- 底部放带宽计算参数框

**演讲者备注：**
"JESD204B是我们方案中最具技术含量的部分之一。传统的LVDS并行接口在2.8Gsps速率下需要大量IO引脚且存在严重的偏移问题。JESD204B将数据串行化到4对差分线上，每对运行在10Gbps，通过8B/10B编码保证DC平衡和时钟恢复。我们采用Subclass 1模式实现确定性延迟，这对于多通道同步输出至关重要——所有4个通道必须精确对齐，相位误差控制在ps级。GTX的配置和调试是整个项目的难点之一，我们将在第11页展开讨论。"

## Slide 7: 核心技术2 — 高精度DDS设计

**内容：**

**左侧 — DDS核心架构：**
```
┌──────────────────────────────────────┐
│           48-bit DDS Core            │
│                                      │
│  FTW ──►┌──────────┐    ┌────────┐  │
│  (48b)  │  48-bit   │───►│  Sine   │  │
│         │  Phase    │    │  LUT    │──┼──► 16-bit
│         │Accumulator│    │(16bit× │  │    DAC Data
│         │           │    │ 2^14)  │  │
│         └──────────┘    └────────┘  │
│              │                       │
│         Phase Offset                 │
│              ↓                       │
│         ┌──────────┐                 │
│         │ Waveform  │                │
│         │  MUX      │──► (方波/三角/ │
│         │           │    锯齿/任意)  │
│         └──────────┘                 │
└──────────────────────────────────────┘
```

**右侧 — 关键参数：**
| 参数 | 数值 | 说明 |
|------|------|------|
| 频率调谐字位宽 | 48 bit | 决定频率分辨率 |
| 系统时钟 f_sys | 700 MHz | 4采样点/输出样点 |
| 有效DAC更新率 | 2.8 Gsps | 实际DAC采样率 |
| 相位累加器位宽 | 48 bit | |
| 频率分辨率 Δf | ~1.24 μHz | f_sys/2^48 |
| SFDR(仿真) | >90 dBc | @低频输出 |
| 波形查找表深度 | 16384 × 16bit | 1/4周期压缩存储 |

**底部 — 波形切换说明：**
- 正弦波：相位累加器 → 1/4周期正弦LUT(利用对称性) → 16-bit输出
- 方波：相位比较器 + 可编程占空比(0.1%~99.9%)
- 三角波：相位到幅度线性映射 + 对称折叠
- 锯齿波：相位直接映射为幅度

**视觉说明：**
- 框图用数字逻辑风格绘制
- 计算公式放大显示: "Δf = f_sys / 2^48 ≈ 1.24 μHz"
- 四种波形右侧配小图标示意

**演讲者备注：**
"我们的DDS核采用48位相位累加器，频率调谐字也为48位。在700MHz系统时钟下，频率分辨率达到约1.24微赫兹——这个精度在低频段意味着可以生成极其平滑的频率扫描。需要注意一个关键设计选择：我们采用4采样/周期的架构，即FPGA内部以700MHz速率生成4倍过采样数据，通过JESD204B送给2.8Gsps的DAC。这样做的好处是：第一，降低FPGA内部逻辑的工作频率到可实现的700MHz；第二，为DAC内部插值滤波器提供高质量的输入数据。正弦波查找表我们使用1/4周期压缩存储，14位地址深度对应16384个采样点，利用对称性恢复完整周期，SFDR仿真结果优于90dBc。"

## Slide 8: 核心技术3 — 时钟系统设计

**内容：**

**左侧 — 时钟树结构：**
```
           ┌─────────────┐
           │  100MHz VCXO │
           │   (Ref In)   │
           └──────┬──────┘
                  │
           ┌──────▼──────┐
           │  LMK04828   │
           │  Dual PLL   │
           │             │
           │ PLL1: VCXO  │────► 内部VCO锁定
           │  Cleanup    │
           │             │
           │ PLL2: VCO   │────► 高频时钟生成
           │  2920~3080  │
           │   MHz       │
           └──┬──────┬──┘
              │      │
    ┌─────────▼┐  ┌──▼──────────┐
    │ DEVCLK   │  │ SYSREF      │
    │ 2.8 GHz  │  │ (可编程分频) │
    │ to DAC   │  │ to FPGA+DAC │
    │ & FPGA   │  │             │
    └──────────┘  └─────────────┘
         │
    ┌────▼─────┐
    │  AD9144  │
    │ DACCLK   │
    │ 2.8GHz   │
    └──────────┘
```

**右侧 — 时钟分配详细参数：**

| 输出 | 频率 | 目标器件 | 用途 |
|------|------|---------|------|
| CLKout0/1 | 2.8 GHz | AD9144 DACCLK | DAC采样时钟 |
| CLKout2/3 | 350 MHz | FPGA GTX REFCLK | JESD204B参考时钟 |
| CLKout4 | 700 MHz | FPGA Logic | DDS系统时钟 |
| SYSREF | 可编程 | FPGA+AD9144 | 确定性延迟同步 |

**SYSREF关键设计要点：**
- 连续SYSREF模式，周期为多帧时钟的整数倍
- SYSREF到DEVCLK的建立/保持时间通过LMK04828内部可编程延迟调节
- 板上走线等长设计，FPGA到DAC的SYSREF skew < 50ps
- 支持SYSREF重定时，消除亚稳态风险

**视觉说明：**
- 时钟树用自上而下的树形图
- 不同频率用不同颜色区分(2.8GHz红色/350MHz蓝色/700MHz绿色)
- 底部放关键时序约束标注

**演讲者备注：**
"时钟系统是高速信号链的灵魂。我们使用TI的LMK04828双PLL时钟芯片。第一级PLL用作抖动清除，锁定100MHz VCXO参考源；第二级PLL的内部VCO运行在2.92-3.08GHz，通过分频器生成所需的各路时钟。最关键的输出是送到AD9144的2.8GHz DAC采样时钟，其相位噪声和抖动直接影响输出信号质量。同时，LMK04828生成的SYSREF信号是实现JESD204B确定性延迟的关键——它定义了系统中所有器件(FPGA和DAC)的共同时间基准。我们在PCB Layout上对关键时钟走线做了严格的等长和阻抗控制，确保多通道间的偏移控制在50ps以内。"

*主要命令集：*
| 命令 | 代码 | 说明 |
|------|------|------|
| SET_FREQ | 0x01 | 设置输出频率(48bit FTW) |

| SWEEP_STOP | 0x05 | 停止扫频 |
| ARB_LOAD | 0x06 | 加载任意波形数据到BRAM |

**区域2 — 本地按键UI（中上）：**
- 4个物理按键：MODE(模式切换) / UP(参数+) / DOWN(参数-) / ENTER(确认)
- 菜单层级：主菜单 → 波形选择 / 频率设置 / 幅度设置 / 扫频设置 / 系统信息
- 长按MODE键3秒进入快速扫频模式

**区域3 — LED状态指示（右侧）：**
- LED1(Green): 系统就绪 — 常亮=正常工作, 闪烁=配置中
- LED2(Blue): JESD204B链路状态 — 常亮=Link Up, 灭=Link Down

**内容：**

**上半部分 — 扫频引擎设计：**

*扫频模式：*
- 线性扫频(Linear Sweep)：频率线性变化，适用于频率响应测试
- 对数扫频(Log Sweep)：频率对数变化，适用于宽频带测试

*扫频参数可编程：*
| 参数 | 位宽 | 说明 |
|------|------|------|
| 起始频率 f_start | 48 bit FTW | 扫频起始点 |
| 终止频率 f_stop | 48 bit FTW | 扫频终止点 |

| 扫频方向 | 1 bit | 0=正向, 1=双向往返 |

*实现方式：* 硬件状态机在FPGA内部实时更新FTW寄存器，无需CPU干预，扫频速率可达微秒级步进间隔

**下半部分 — BRAM任意波形回放：**

*存储架构：*
- Block RAM：4096 × 16 bit × 4通道 (独立)
- 波形数据通过UART的ARB_LOAD命令下载到BRAM
- 支持乒乓缓冲：写入BRAM Bank A时从Bank B读取输出，实现无缝切换

*波形切换机制：*
```
波形选择 → 波形MUX → ┬─ 正弦DDS输出
                      ├─ 方波生成器输出
                      ├─ 三角波生成器输出
                      ├─ 锯齿波生成器输出
                      └─ BRAM回放输出
                                │
                          16bit DAC Data
```
切换时在波形过零点处执行，避免输出毛刺

## Slide 12: 关键技术难点与解决方案

**内容（四栏或四行布局）：**

---

**难点1：GTX收发器在10.3125Gbps下的信号完整性**

- **问题描述：** 10Gbps以上高速串行信号对PCB走线、连接器、阻抗匹配极其敏感，信号衰减和码间干扰严重
- **解决措施：**
  - PCB差分走线严格100Ω差分阻抗控制，线长匹配<5mil
  - GTX TX Pre-emphasis参数经过IBIS-AMI仿真优化，前标加重+3.5dB
  - 接收端AD9144内置CTLE(连续时间线性均衡)+DFE(判决反馈均衡)
  - 眼图仿真验证：水平张开度>0.7UI @ BER 1e-15
- **结果：** 链路稳定建立，无误码运行

---

**难点2：JESD204B多链路确定性延迟对齐**

- **问题描述：** 4个通道输出的相对相位必须精确对齐，SYSREF分发和链路延迟补偿是JESD204B中最复杂的部分
- **解决措施：**
  - 采用Subclass 1模式实现确定性延迟
  - LMK04828 SYSREF走线严格等长(长度差<5mil)
  - 利用AD9144内部LMFC缓冲和RBD(接收缓冲延迟)调节
  - 通过校准流程测量各通道相对延迟，在FPGA TX侧补偿
- **结果：** 4通道间相位误差<50ps(相当于2.8GHz下一个采样周期的14%)

---

**难点3：DDS高速流水线时序收敛**

- **问题描述：** 700MHz FPGA内部逻辑时钟下，48位加法器+查找表访问的组合路径难以满足时序约束
- **解决措施：**
  - 相位累加器分成3级流水线(16+16+16 bit)，每级延迟1个时钟周期
  - 正弦LUT使用双端口BRAM，流水线化读取
  - 4采样/周期的并行处理架构，将关键路径从700MHz放松到175MHz等效处理
  - 使用Vivado物理优化(PhysOpt)反复迭代布局布线
- **结果：** 建立时间裕量>0.2ns，保持时间裕量>0.15ns

---

**难点4：电源完整性 — 高速混合信号系统的噪声隔离**

- **问题描述：** FPGA数字开关噪声和DAC模拟输出的相互干扰，直接影响DAC的SFDR
- **解决措施：**
  - 数字电源和模拟电源完全独立的两组LDO供电网络
  - AD9144 AVDD使用超低噪声LDO(噪声密度<5μV/√Hz)
  - PCB采用完整地层分割，数字地与模拟地在DAC下方单点连接
  - 关键模拟走线周围加屏蔽地层
- **结果：** 实际测试DAC输出底噪<-145dBm/Hz

---

- 四个难点用带编号的卡片式布局，每个卡片包含问题图标(红色)、方案图标(绿色)、结果图标(蓝色)
- 或使用四列布局，每列从上到下：难题名称 → 问题 → 方案 → 结果

**演讲者备注：**
"这四个难点反映了高速混合信号系统中典型的工程挑战。第一个难点是物理层的信号完整性问题——10Gbps已经进入微波频段，PCB走线就是传输线。第二个难点是JESD204B协议层的同步问题，这在多通道系统中尤为关键。第三个难点是数字设计中的时序收敛，700MHz在FPGA中属于较高频率。第四个难点是大多数信号链项目中容易被忽视但至关重要的电源完整性问题。每个难点我们都有针对性的解决方案和实测数据支撑，下面进入测试验证部分。"

## Slide 16: 指标差距分析

**内容：**

**核心差距：采样率 2.8 Gsps vs 赛题要求 5 Gsps**

**AD9144 选型论证：**

| 考量维度 | 分析 |
|---------|------|
| **经济可行性** | 5Gsps级DAC(AD9164/AD9174)单芯片$500+且供货不稳定，竞赛经济约束下不现实 |
| **垂直分辨率优势** | AD9144提供16bit物理位宽，超过赛题14bit要求，优于大多数5Gsps DAC(12-14bit) |
| **4通道能力** | 四路独立输出，支持多通道同步测试场景 |
| **架构可迁移性** | JESD204B框架和48bit DDS核可直接迁移至更高端平台 |

**多维度补偿策略：**

1. **垂直分辨率超标：** 16bit vs 14bit，ENOB每高1bit动态范围+6dB
2. **通道数超额：** 4通道独立输出
3. **频率分辨率极高：** ~μHz级，远超1mHz要求
4. **系统完整度：** 完整的JESD204B协议栈 + 自研扩展模块(显示屏+旋钮+PCB丝印)
5. **自研PCB：** KiCad设计的扩展模块满足Logo丝印要求，提供本地交互能力

**实事求是：**
我们的方案在采样率这一单项指标上未达到赛题5Gsps要求。但综合考虑垂直分辨率、通道数、频率分辨率和系统功能完整度，方案在整体技术价值上具有竞争力。采样率的差距我们向评委坦诚说明，这是基于现实约束的理性工程决策。

**视觉说明：**
- 顶部用大号数字对比：2.8 Gsps vs 5 Gsps，但不使用红色(避免负面强调)
- 中间四行分析用表格/矩阵
- 底部弥补措施用对勾列表
- 整体色调保持专业和自信

**演讲者备注：**
"这一页我们坦诚地讨论方案中唯一的明显差距。我们的策略是'以精度换速度'——在可获得的器件平台上做到极致的频率分辨率和通道数，同时保证系统级的功能完整。我们相信评委更看重的是一个完整、可工作、技术路线清晰的系统，而不是追求纸面参数的最大化。而且我们的JESD204B架构本身支持向更高速率迁移——相同的设计方法学可以直接复用到下一代5G DDS方案中。"
