# 时钟树

## 系统时钟源

```
                    ┌─────────────┐
    50M TCXO        │  LMK04828   │        输出分配
   (0.3ppm)    ───→ │  (PLL+分频) │  ───┬──→ OUT0 = 125M  → FPGA GTX QPLL0_REFCLK
                    │             │     ├──→ OUT8 = 500M   → AD9144 DACCLK
                    │  SPI配置    │     ├──→ OUT1 = 3.90625M → SYSREF (Subclass1)
                    │  (lmk04828  │     └──→ 其他 = ...
                    │   _spi_ctrl)│
                    └─────────────┘
```

## FPGA 内部时钟域

| 时钟域 | 频率 | 来源 | 用途 |
|---|---|---|---|
| `clk` | 100 MHz | sys_clk_p/n (IBUFDS+BUFG) | 系统主时钟、SPI、状态机 |
| `gtx_refclk` | 125 MHz | LMK OUT0 → GTX IBUFDS_GTE2 | GTX 收发器参考时钟 |
| `dac_clk` | 250 MHz | gtx_refclk × 2 (PLL) | JESD204 link clock (lane_rate/40) |
| `sysref` | 3.90625 MHz | LMK OUT1 → FPGA | LMFC 对齐、确定性延迟 |

## JESD204 时钟关系

```
Lane Rate = 10 Gbps
Character Clock = Lane Rate / 40 = 250 MHz  (8B/10B)
Frame Clock = Character Clock / F = 250 MHz  (F=1)
Sample Clock = Frame Clock / S = 250 MHz     (S=1)
LMFC = Sample Clock / K = 250M / 32 = 7.8125 MHz
SYSREF = 3.90625 MHz (LMFC / 2, gapped periodic)
```

## 时钟约束要点

```tcl
# 100M 系统时钟
create_clock -period 10.000 -name sys_clk [get_ports sys_clk_p]

# GTX 参考时钟 (125M)
create_clock -period 8.000 -name gtx_refclk [get_ports fmc_gbtclk0_p]

# SYSREF (3.90625M, 源同步)
create_clock -period 256.000 -name sysref [get_ports fmc_sysref_p]

# 异步时钟域 crossing
set_false_path -from [get_clocks sys_clk] -to [get_clocks gtx_refclk]
set_false_path -from [get_clocks gtx_refclk] -to [get_clocks sys_clk]
```

## 注意事项

1. **LMK04828 必须先锁定**，FPGA 才能开始 JESD204 建链
2. **SYSREF 必须是 gapped periodic**（不是连续时钟），每次只发一个脉冲对齐 LMFC
3. **所有时钟必须同源**（来自 LMK04828），保证整数倍关系
4. GTX 参考时钟推荐使用 **QPLL**（而非 CPLL），因为 lane rate 10Gbps 接近 GTX 上限
